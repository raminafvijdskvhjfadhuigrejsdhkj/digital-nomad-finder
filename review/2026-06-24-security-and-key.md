# Итерация 1 — безопасность и базовый React (2026-06-24)

## [1] API-ключ зашит прямо в код (важно!)
- **Файл:** `src/App.jsx:13`
- **Что:** токен доступа к API (`Authorization: "Bearer rc_live_..."`) написан
  прямо в исходном коде и уже закоммичен в git.

- **Почему это серьёзно:**
  1. **История git вечна.** Даже если сейчас удалить строку и сделать новый
     коммит — старый коммит с ключом останется в истории. Любой, у кого есть
     доступ к репозиторию (а он на GitHub), сможет достать ключ из прошлых
     коммитов. Просто «стереть и закоммитить» — НЕ решает проблему.
  2. **Фронтенд-код виден всем.** Даже без git: всё, что попадает в браузерный
     бандл, пользователь может прочитать через DevTools. Секретам не место
     в клиентском коде в принципе.
  3. **Утечка = чужие расходы/блокировка.** По такому ключу могут слать запросы
     от твоего имени — исчерпать лимиты, обрушить квоту или подставить под
     блокировку аккаунт.

- **Как двигаться (попробуй по шагам):**
  1. **Считай этот ключ скомпрометированным** — он уже в истории на GitHub.
     Правильный первый шаг в реальной работе — выпустить новый ключ (rotate),
     а старый отозвать. Если это твой тестовый ключ — отзови его в кабинете API.
  2. Вынеси ключ из кода в переменную окружения Vite: файл `.env` в корне,
     переменная с префиксом `VITE_` (без него Vite её не отдаст в код).
     Читается через `import.meta.env`.
  3. Добавь `.env` в `.gitignore`, чтобы он никогда не попал в git. Рядом полезно
     положить `.env.example` с именами переменных без значений — чтобы другой
     разработчик знал, что нужно завести.
  4. Подумай над вопросом для разбора: а можно ли вообще безопасно держать
     «секрет» во фронтенде? (Подсказка: по-настоящему секретные ключи прячут
     за своим бэкендом-прокси. `.env` во фронте лишь убирает ключ из кода,
     но не из бандла.)

## [2] Неверный `key` в списке стран
- **Файл:** `src/pages/Home.jsx:178`
- **Что:** `key={country.name?.common}` — поле называется `name`, а в данных оно
  `names` (см. `App.jsx` и `CountryCard.jsx`, где везде `country.names`).
  Приложение работает и страны рендерятся, но `key` получается `undefined`.
- **Почему:** в консоли висит ошибка React «Each child in a list should have a
  unique key prop». Без корректного `key` React не может эффективно обновлять
  список — это приводит к багам при фильтрации/сортировке.
- **Как:** используй уникальное и стабильное значение — например `country.names.common`
  (как в остальном коде).
