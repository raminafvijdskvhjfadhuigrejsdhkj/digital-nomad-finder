## Замечания

### [1] API-ключ зашит прямо в код
- **Файл:** `src/App.jsx:13`
- **Что:** токен `Bearer rc_live_...` написан прямо в коде.
- **Почему:** всё, что попадает в git, остаётся в истории навсегда — даже если потом
  удалить. Секреты так утекают. Плюс ключ нельзя поменять без правки кода.
- **Как:** посмотри в сторону переменных окружения Vite (`.env` + `import.meta.env`).
  Файл `.env` добавляется в `.gitignore`.

### [2] Несостыковка структуры данных от API
- **Файлы:** `src/App.jsx:18` и `src/pages/Home.jsx:178`
- **Что:** в `App` страны читаются как `country.names.common`, а в карточке —
  как `country.name?.common`. Поля называются по-разному.
- **Почему:** из-за этого список, скорее всего, не отрендерится — `key` будет `undefined`.
- **Как:** проверь реальный ответ API (console.log или вкладка Network) и приведи
  обращения к данным к одному виду.

### [3] Не зафиксирована версия Node
- **Файл:** `package.json`
- **Что:** нет поля `engines` и файла `.nvmrc`.
- **Почему:** на разных версиях Node/npm проект может вести себя по-разному
  (например, у нас уже «прыгал» `package-lock.json`).
- **Как:** добавь `.nvmrc` с версией Node или поле `engines` в `package.json`.
